# 1.1 迷你交易所

## 为什么做这个项目？

你每次在券商 App 上点「买入」，背后都有一个撮合引擎在工作。它决定了你的订单以什么价格、在什么时候成交。理解这个引擎，就是理解市场最底层的运作逻辑。

这个项目让你**自己写一个简化版的交易所**。当你亲手实现了订单撮合、看到价格如何从买卖双方的博弈中「涌现」出来，你对市场的理解会发生质变。

## 已知领域（你已经会的）

- Python 基础：类、列表、字典、排序
- 你作为交易者的直觉：你知道什么是买入、卖出、限价

## 新领域（你需要学的）

- 订单簿（Order Book）的数据结构
- 撮合算法：价格优先、时间优先
- 市价单 vs 限价单的本质区别
- 流动性和价格冲击的概念

---

## Task 1: 理解订单簿，实现限价单数据模型

**做什么**: 设计一个 `OrderBook` 类，能够接收限价买单和卖单，并按规则组织存储。

**思考提示**:
- 买方出价（bid）和卖方要价（ask）分别应该按什么顺序排列？为什么？
- 当买一价和卖一价之间存在间隙（spread），这个间隙代表什么经济含义？
- 为什么不能用一个简单的列表来存储订单？什么数据结构更合适？

**验证**: 你能往订单簿里添加多个买单和卖单，打印出来后买方按价格从高到低排列，卖方按价格从低到高排列。

📚 **关键资料**:
- 搜索「Order Book」和「Limit Order Book」的概念，理解 bid/ask/spread
- 了解 Python 的 `heapq` 模块或 `sortedcontainers` 库，思考哪个适合做订单簿

---

## Task 2: 实现撮合引擎核心逻辑

**做什么**: 当一个新订单进入时，检查是否能与对手方订单成交。实现「价格优先、时间优先」的撮合规则。

**思考提示**:
- 一个大买单可能需要吃掉多个卖单才能完全成交，这叫什么？你的撮合逻辑怎么处理这种情况？
- 如果买方出价 10.5 但最优卖方要价是 10.3，成交价应该是多少？为什么？
- 部分成交的订单应该怎么处理？删掉？修改？

**验证**: 提交一系列买卖订单后，撮合引擎正确产生成交记录（成交价、成交量、买卖双方），未成交部分留在订单簿中。

📚 **关键资料**:
- 搜索「price-time priority matching」了解撮合规则
- 思考为什么全球几乎所有交易所都用这个规则（有没有例外？）

---

## Task 3: 支持市价单，观察价格冲击

**做什么**: 在限价单的基础上，增加对市价单的支持。市价单不指定价格，以当前最优价格立即成交。

**思考提示**:
- 如果订单簿上只有 100 股在卖，但有人下了 500 股的市价买单，会发生什么？
- 「价格冲击」（price impact）是什么意思？一个大单为什么会推动价格？
- 为什么机构投资者通常不会直接下大额市价单？他们会怎么做？

**验证**: 在一个流动性较低的订单簿上提交大额市价单，观察到价格被逐级推高（或压低），并且成交均价明显偏离初始最优价。

📚 **关键资料**:
- 搜索「market impact」和「slippage」的概念
- 了解冰山订单（iceberg order）和 TWAP/VWAP 算法的基本思想

---

## Task 4: 模拟多个交易者，观察价格形成

**做什么**: 创建多个模拟交易者（用简单的随机策略即可），让他们不断提交订单，观察价格的变化轨迹。

**思考提示**:
- 如果所有交易者都是随机的，价格应该呈现什么样的走势？（提示：想想随机游走）
- 如果你加入一个「趋势跟踪者」——价格涨了就买、跌了就卖——价格走势会怎么变化？
- 如果你加入一个「做市商」——在买一和卖一之间同时挂单赚价差——市场会怎么变化？
- 这些模拟结果和真实市场有什么相似和不同？

**验证**: 运行模拟后生成一条价格时间序列，并且你能用文字解释为什么价格呈现这样的走势。

📚 **关键资料**:
- 搜索「random walk hypothesis」和「agent-based model」
- 了解做市商（market maker）的基本概念和盈利模式

---

## Task 5: 可视化订单簿深度和成交记录

**做什么**: 用 matplotlib 绘制两种图表：(1) 订单簿深度图（买方/卖方累积挂单量 vs 价格）；(2) 成交价格的时间序列。

**思考提示**:
- 在真实交易软件中，你见过类似订单簿深度图吗？它传递了什么信息？
- 如果深度图中卖方在某个价位堆积了大量订单（「卖墙」），这意味着什么？价格一定过不去吗？
- 你的成交价时间序列和真实的股票 K 线有什么区别？缺少了什么信息？

**验证**: 生成的图表清晰可读，你能用它们向别人解释订单簿的工作原理。

## 完成标准

- **行为验证**: 你的迷你交易所能正确处理限价单和市价单的撮合，多个模拟交易者能同时运行
- **解释验证**: 你能向一个完全不懂交易的人解释：为什么股票有「买一价」和「卖一价」，以及它们之间的差距意味着什么
- **迁移验证**: 当你看到加密货币交易所的订单簿界面时，你能指出其中的每一个组成部分，并解释它的作用

## 延伸思考

在真实交易所中，高频交易（HFT）公司就是在这个你刚刚实现的撮合层面上竞争的。他们追求的是微秒级的速度优势。思考一下：这对普通投资者意味着什么？如果你无法在速度上和他们竞争，你应该在哪个维度上建立你的优势？

这个问题的答案，会在 Chapter 3（策略）和 Chapter 4（风险管理）中逐渐清晰。
