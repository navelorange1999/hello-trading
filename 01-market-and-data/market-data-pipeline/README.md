# 1.2 市场数据管道

## 为什么做这个项目？

数据是量化交易的燃料。没有可靠的数据，后面所有的分析、回测、策略都是空中楼阁。

但「获取数据」远不是调一下 API 那么简单。真实的市场数据充满了陷阱：停牌导致的缺失值、拆股导致的价格跳变、不同数据源之间的不一致。这个项目让你从头构建一个**可靠的数据管道**，这将是你整个学习路线的基础设施。

## 已知领域

- 你知道什么是股票价格、K 线图（至少在券商 App 上看过）
- Python 基础和 Chapter 1.1 中获得的信心

## 新领域

- 通过 API 获取金融数据
- 数据清洗：处理缺失值、异常值、复权
- OHLCV（开盘、最高、最低、收盘、成交量）数据的含义
- 基本的统计描述：收益率、波动率、相关性
- 用代码绘制 K 线图

## Task 1: 获取股票和加密货币的历史 OHLCV 数据

**做什么**: 使用 Python 库从公开 API 获取至少两只股票和一种加密货币的日线历史数据。

**思考提示**:
- OHLCV 中的每一个字母代表什么？为什么不是只记录收盘价，还要记录开盘、最高、最低？
- 日线数据和分钟线数据的区别不只是粒度——想想看，1 分钟 K 线的「开盘价」意味着什么？
- 免费数据源和付费数据源的差异在哪里？对你当前的学习来说，这个差异重要吗？

**验证**: 成功获取数据并打印前 10 行，每行包含日期、开盘价、最高价、最低价、收盘价、成交量。

📚 **关键资料**:
- 搜索 "yfinance python" — 获取美股/港股数据的最简单方式
- 搜索 "ccxt python" — 统一接口获取各大加密货币交易所数据
- 搜索 "akshare" 或 "tushare" — 获取 A 股数据的选择

---

## Task 2: 数据清洗与持久化存储

**做什么**: 检查下载的数据中是否存在缺失值、异常值。处理停牌日的缺失数据。理解并实现前复权/后复权。将清洗后的数据存储到本地文件。

**思考提示**:
- 如果某天的数据缺失（比如停牌），你应该怎么处理？填充？删除？不同的选择会如何影响后续分析？
- 一只股票 10 送 10 后，价格从 100 变成了 50，但公司价值没变。如果不做复权处理，你的分析会出什么问题？
- CSV、SQLite、Parquet——存储市场数据各有什么优劣？对你当前的需求来说，哪个最实用？

**验证**: 清洗后的数据没有缺失值，复权价格连续平滑，数据已持久化到本地文件，重启程序后可直接加载。

📚 **关键资料**:
- 搜索 "stock data adjusted price vs unadjusted" — 理解复权的必要性
- 搜索 "pandas fillna methods" — 了解不同的缺失值处理方法
- 搜索 "python sqlite3 tutorial" 或 "pandas to_parquet" — 数据持久化方案

---

## Task 3: 从零绘制 K 线图

**做什么**: 不使用 mplfinance 或其他现成的 K 线绘图库，用 matplotlib 的基础绑图功能（矩形、线段）自己画一张 K 线图。

**思考提示**:
- K 线图上的一根「蜡烛」到底编码了哪些信息？一根红色（阴线）长上影线的蜡烛告诉你市场发生了什么？
- 为什么 K 线图比折线图（只画收盘价）更受交易者欢迎？它多传达了什么信息？
- 成交量应该怎么叠加到 K 线图上？为什么价格和成交量的关系很重要？

**验证**: 生成的 K 线图在视觉上与券商 App 中的日 K 线图一致，包含蜡烛体、上下影线、成交量柱。

📚 **关键资料**:
- 搜索 "matplotlib patches Rectangle" — 用矩形画蜡烛体
- 搜索 "candlestick chart anatomy" — 确保你理解了每个组成部分
- 把你画的图和券商 App 的图对比，找到差异

## Task 4: 计算基础统计量

**做什么**: 对下载的数据计算：日收益率、累计收益率、年化波动率、不同资产之间的相关系数。

**思考提示**:
- 日收益率应该用简单收益率 `(P1-P0)/P0` 还是对数收益率 `ln(P1/P0)`？两者有什么区别？什么时候用哪个？
- 年化波动率 20% 到底意味着什么？一只年化波动率 20% 的股票和一只 60% 的，你持有它们的体验有什么不同？
- 两只股票的相关系数是 0.9 意味着什么？如果是 -0.3 呢？这对你的资产配置有什么启发？

**验证**: 能产出一个简洁的统计摘要，包含每只资产的年化收益率、年化波动率，以及资产间的相关性矩阵。

📚 **关键资料**:
- 搜索 "log return vs simple return finance" — 理解两种收益率计算方式
- 搜索 "annualized volatility formula" — 理解日波动率如何转换为年化
- pandas 的 `.pct_change()`, `.std()`, `.corr()` 方法

## Task 5: 封装为可复用的数据模块

**做什么**: 将前面的代码整理为一个模块，提供清晰的接口：给定股票代码和日期范围，返回清洗好的 DataFrame。

**思考提示**:
- 好的 API 设计是什么样的？如果另一个人要用你的模块，他需要知道多少内部细节？
- 如果数据源的 API 挂了或者返回格式变了，你的模块应该怎么处理？
- 这个模块在后续的每一章都会用到——想想看，你还需要添加什么功能来满足未来的需求？

**验证**: 模块能被 import 使用，两行代码即可获取一只股票的干净数据。后续项目实际引用了这个模块。

## 完成标准

- **行为验证**: 用你的模块获取任意一只股票或加密货币的数据，5 秒内得到干净的 DataFrame 和 K 线图
- **解释验证**: 你能解释复权的必要性、波动率的含义、以及为什么相关性对投资组合很重要

## 延伸思考

你现在用的是日线数据，这对于「睡后收入」型的中长期策略足够了。但如果有一天你想做更短周期的交易（小时级甚至分钟级），你的数据管道需要做哪些改变？数据量会增大多少倍？存储和处理方式需要怎么调整？

这些问题不需要现在回答，但在心里留个印象——它们会在 Chapter 5 变得相关。
