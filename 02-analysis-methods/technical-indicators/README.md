# 2.1 技术指标实验室

## 为什么做这个项目？

打开任何一个交易软件，你都会看到一堆花花绿绿的线条和指标。很多人用它们交易，但大多数人并不真正理解这些指标在**数学上**做了什么。

「MACD 金叉买入」——为什么？金叉的本质是什么？它在什么条件下有效、什么条件下失效？如果你不能回答这些问题，那你就是在盲目遵循一个你不理解的规则。

这个项目的核心不是「学会用技术指标」，而是**从零实现每一个指标，理解它的数学本质，然后用数据检验它是否真的有用**。

## 已知领域

- Chapter 1 的数据管道（获取和处理 OHLCV 数据）
- 基本的 pandas 操作

## 新领域

- 移动平均、指数加权的数学含义
- 动量指标 vs 趋势指标的区别
- 统计假设检验的基本思想
- 如何区分「看起来有效」和「统计上显著」

## Task 1: 从零实现移动平均线（SMA、EMA）

**做什么**: 不使用 pandas 的 `.rolling().mean()` 或 `.ewm()`，用基础循环自己实现 SMA 和 EMA 的计算。实现之后再用 pandas 内置方法验证结果一致。

**思考提示**:
- SMA(20) 的值在数学上等于什么？它为什么叫「移动」平均？
- EMA 和 SMA 的核心区别是什么？EMA 的「指数」体现在哪里？
- 为什么 EMA 对近期价格更敏感？这种敏感性是优点还是缺点？取决于什么？
- 短期均线（如 5 日）和长期均线（如 200 日）各自在告诉你什么信息？

**验证**: 你的手动实现和 pandas 内置方法的结果完全一致（误差 < 0.0001）。你能用自己的话解释 EMA 的权重衰减机制。

📚 **关键资料**:
- 搜索 "exponential moving average formula step by step" — 先理解公式再写代码
- EMA 的平滑因子 α = 2/(N+1) 是怎么来的？搜索 "EMA smoothing factor derivation"

## Task 2: 实现 RSI 和 MACD

**做什么**: 从零实现 RSI（相对强弱指数）和 MACD（指数平滑异同移动平均线）。

**思考提示**:
- RSI 的值域是 0-100，它衡量的到底是什么？「超买」和「超卖」的本质含义是什么？
- RSI 和价格涨跌幅有什么关系？一只连涨 14 天的股票，RSI 一定是 100 吗？
- MACD 由三部分组成（DIF、DEA、柱状图），它们各自反映了什么信息？
- MACD 的「金叉」和「死叉」，从数学上来说，到底在告诉你什么？（提示：想想两条 EMA 的相对位置变化）

**验证**: 用 一只股票的真实数据计算 RSI 和 MACD，与主流交易软件的指标值对比，数值一致。

📚 **关键资料**:
- 搜索 "RSI calculation method Wilder" — 注意 RSI 有不同的计算方式，先了解 Wilder 的原版
- 搜索 "MACD indicator explained" — 重点理解它是两条 EMA 之差，而不是一个独立的计算

## Task 3: 实现布林带

**做什么**: 实现布林带（Bollinger Bands）：中轨 = SMA(20)，上轨 = 中轨 + 2σ，下轨 = 中轨 - 2σ。

**思考提示**:
- 布林带的宽度在扩大和收窄时，分别意味着市场在做什么？
- 为什么用 2 倍标准差？如果假设收益率服从正态分布，价格落在布林带内的概率是多少？
- 实际上，股票收益率真的服从正态分布吗？如果不是（提示：查一下「肥尾分布」），这对布林带的解读有什么影响？
- 布林带和前面学的 RSI 有没有关联？它们在什么情况下会给出相同/矛盾的信号？

**验证**: 布林带正确绑在 K 线图上，带宽在市场波动加剧时扩大、平静时收窄。

📚 **关键资料**:
- 搜索 "Bollinger Bands calculation and interpretation"
- 搜索 "fat tails in stock returns" — 理解为什么正态分布假设在金融中是危险的

## Task 5: 用数据验证一个常见说法

**做什么**: 选一个常见的技术分析说法（如「MACD 金叉买入胜率高于 50%」或「RSI 跌破 30 后买入能赚钱」），用历史数据进行统计验证。

**思考提示**:
- 你怎么定义「有效」？赚钱了就算有效吗？如果赚的比大盘少呢？
- 样本量够大吗？如果 10 年里只出现了 5 次金叉，你的结论可靠吗？
- 如果某个信号在牛市中很准、在熊市中很差，平均下来「还行」，这个平均数有意义吗？
- 这个验证过程和科学实验的「假设检验」有什么相似之处？

**验证**: 你产出了一份有数据支撑的小报告，包含：检验的假设、样本数据、统计结果、你的结论。结论不需要是「有效」——「这个说法在我的数据中不成立」也是有价值的发现。

## 延伸思考

你在 Task 5 中可能发现，单个技术指标的预测能力并不强。这其实是学术界的共识——大量研究表明，简单的技术分析规则很难持续盈利。

那为什么还要学？两个原因：
1. 这些指标是量化策略的「词汇表」——更复杂的策略会组合使用它们
2. 理解它们的局限性本身就是知识——它帮你避免了盲目相信技术分析的陷阱
